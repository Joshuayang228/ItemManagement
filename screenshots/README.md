## 🎨 截图美化建议

### 数据准备
1. **添加示例物品**：至少 10-15 个不同类别的物品
2. **使用真实图片**：避免使用占位图或模糊图片
3. **设置不同状态**：
   - 部分物品快过期（显示提醒标签）
   - 部分物品有地理位置
   - 部分物品有多张图片

### 界面美化
1. **关闭开发者选项**：隐藏布局边界等调试信息
2. **清理通知栏**：截图前清理通知
3. **选择合适时间**：显示真实的系统时间（如 10:00、14:30）
4. **电量充足**：截图时电量建议在 60% 以上

### 截图技巧
1. **使用系统截图**：
   - 同时按 `电源键` + `音量下键`
   - 或使用三指下滑截图（部分设备）

2. **使用 Android Studio**：
   - 连接真机或模拟器
   - 点击工具栏的 📷 相机图标
   - 保存到本地

3. **使用 ADB**：
   ```bash
   adb shell screencap -p /sdcard/screenshot.png
   adb pull /sdcard/screenshot.png
   ```

## 📝 截图后处理

### 文件命名
按照上面的清单命名：
- `home.png` - 主界面
- `detail.png` - 物品详情
- `map.png` - 地图选点
- `shopping.png` - 购物清单
- `analysis.png` - 数据分析

### 图片优化
1. **压缩大小**：使用工具压缩到 500KB 以内
   - 在线工具：[TinyPNG](https://tinypng.com/)
   - 桌面工具：ImageOptim (Mac) / FileOptimizer (Windows)

2. **裁剪边缘**：移除黑边和多余空白

3. **添加设备框**（可选）：
   - 使用 [Mockuphone](https://mockuphone.com/)
   - 或 [Smartmockups](https://smartmockups.com/)

## ❓ 常见问题

**Q: 截图太大无法上传到 GitHub？**
A: 使用 TinyPNG 等工具压缩，GitHub 单文件限制 25MB，一般截图压缩后都在 1MB 以内。

**Q: 不想显示真实数据怎么办？**
A: 可以添加测试数据，使用假名称和模糊的地理位置。

**Q: 截图中能出现个人信息吗？**
A: 避免出现真实姓名、地址、手机号等隐私信息。

**Q: 需要截图深色模式吗？**
A: 建议提供 1-2 张深色模式截图，展示主题适配效果。
